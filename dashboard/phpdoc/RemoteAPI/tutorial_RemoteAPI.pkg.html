RemoteAPI

Remote XML API

An XML report of a website's environment is available through HTTP requests. No password is required (to facilitate programmatic usage) but it is restricted to select IP addresses (generally the EuPathDB servers).

General Usage For the full report document, make a request to /dashboard/xml. $ curl 'http://toxodb.org/dashboard/xml' 

<?xml version="1.0"?>
<dashboard version="1.0">
  <proxy>
    <proxyapp>nginx</proxyapp>
    <proxyhost>128.192.75.110</proxyhost>
    <upstreamhost>w2.toxodb.org</upstreamhost>
  </proxy>
  <wdk>
    <modelname>ToxoDB</modelname>
    <modelversion>6.3</modelversion>
    <databases>
      <appdb>
        <servicename>TOXO630N.pcbi.upenn.edu</servicename>
        <instancename>toxo630n</instancename>
        <globalname>TOXO630N.PCBI.UPENN.EDU</globalname>
        <aliases>
          <alias>toxo630n</alias>
        </aliases>
      </appdb>
      <userdb>
        <servicename>apicomm.pcbi.upenn.edu</servicename>
        <instancename>apicomm</instancename>
        <globalname>APICOMM.PCBI.UPENN.EDU</globalname>
        <aliases>
          <alias>apicommn</alias>
        </aliases>
      </userdb>
    </databases>
  </wdk>
</dashboard>

You can use a sub-path, e.g. /dashboard/xml/servicename, and it returns first match. Be sure the path is unabiguous. For example, /dashboard/xml/servicename is ambiguous for /dashboard/xml/userdb/servicename vs /dashboard/xml/appdb/servicename

The default is to return data as XML. 

$ curl http://toxodb.org/dashboard/xml/appdb/servicename
<?xml version="1.0"?>
<servicename>TOXO640N.pcbi.upenn.edu</servicename>

Append the /value keyword to get plain text. 
$ curl http://toxodb.org/dashboard/xml/appdb/servicename/value
TOXO640N.pcbi.upenn.edu

Ordering is not guaranteed to be consistent across code updates.

If you don't find a match for a path that you know should be correct, check the full document for 

<error>
nodes.

There is a special-case URI, /dashboard/xml/version that returns the version of the report.

Technical

php-5.x and the php-xml RPM packages are required on RedHat systems.

The /dashboard web interface has writable components (e.g. annoucements), so it is restricted by IP and password. The xml api /dashboard/xml is read only, so is IP restricted only. To aid scripted use, password is not required. Be extra careful that no passwords or other sensitive data is returned in the xml.

The version attribute at root is for the format and elements returned, 

<dashboard version='1.0'>
If the format changes (elements added, removed), change the version in the PHP script.

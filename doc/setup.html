The GUS Build System


INSTALLATION

1. See if you have Ant installed:
% ant -help
  ant [options] [target [target2 [target3] ...]]
   Options:
     -help                  print this message
     -projecthelp           print project help information

   If not, download Ant and unpack it.


2. See if your $JAVA_HOME variable is set:
% echo $JAVA_HOME
/usr/java/j2sdk1.4.0/

    If not, find out where your jdk is installed.


3. Set up a directory which will house your projects.  (This is where you will check out from cvs, and where you will edit the files you are developing.)  Wherever you decide to place this directory, set PROJECT_HOME to it:
% mkdir $HOME/projects

4. Set up a directory from which you will run GUS and related software.  (This is the target of the build program):
% mkdir $HOME/gus_home

5. Set up your environment by placing the following in your .login or .cshrc (using the proper syntax for your shell):
# GUS build system
setenv ANT_HOME /usr/local/pkg/ant/jakarta-ant-1.5   (ie, where YOUR Ant is installed)
setenv JAVA_HOME /usr/java/j2sdk1.4.0/               (ie, where YOUR Java is installed)
setenv PROJECT_HOME $HOME/projects
setenv GUS_HOME $HOME/gushome
setenv GUS_CONFIG_FILE $HOME/.gus.properties
setenv PATH $GUS_HOME/bin:$PROJECT_HOME/install/bin:$ANT_HOME/bin:${PATH}

6. Make your new environment take effect by starting a new terminal session (or by using source and rehash).


7. Check out the GUS and install projects from the public GUS CVS repository.
if you are a GUS developer:
% cd $PROJECT_HOME
% cvs -d :ext:cvs.sanger.ac.uk:/cvsroot/GUS co install GUS
otherwise, go to the publicly available cvs website http://cvsweb.sanger.ac.uk/

8. Get other CBIL projects.  At a minimum you need the CBIL project because GUS depends upon it..  You may also get other CBIL projects as needed.  
If you are not at CBIL, get them from the CBIL download site:
  • the CBIL project (get the latest release)
  • other projects
then install them:
% cd $PROJECT_HOME
% tar -xf CBIL-x.x.x.tar  
If you are at CBIL, get them from the internal CBIL CVS:
% cd $PROJECT_HOME
% cvs -d /ptolemy/cbil/cvs/cvsroot co CBIL ...

9. Set up the GUS config file.
% cp $PROJECT_HOME/install/gus.properties.sample $GUS_CONFIG_FILE
    Edit the config file to reflect your personal information.


10. Give the build system a try:
% build
usage:
  build AllGenes|Annotator|CBIL|DJob|DoTS|GUS|ParaDBs|PlasmoDB[/componentname]
install|installweb|package  targetDir -append [-co [version]]

% build GUS install -append
